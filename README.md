# Aether

A SwiftUI iOS app for jamming progressive / melodic house from a phone — a simple
"Vital-style" wavetable synth wrapped in a loop-based **stack of layers**
(Bass → Chords → Drums → Lead). Built from the `design_handoff_aether_app/` spec
("Aether Stacks" direction).

## Status

All 8 screens implemented to the high-fidelity spec, plus a **real audio engine**
(not mocked): a polyphonic wavetable synth, per-layer filters, a BPM transport that
sequences bass/chords/drums, live scale-locked lead pads, and a shimmer reverb.

## Build & run

```sh
xcodegen generate        # regenerate Aether.xcodeproj from project.yml
open Aether.xcodeproj     # ⌘R on an iPhone simulator/device
```

Or from the CLI:

```sh
xcodebuild -project Aether.xcodeproj -scheme Aether \
  -sdk iphonesimulator -destination 'platform=iOS Simulator,name=iPhone 17 Pro' build
```

> `Aether.xcodeproj` is generated by [xcodegen](https://github.com/yonyz/xcodegen)
> and git-ignored — the source of truth is `project.yml`. Run `xcodegen generate`
> after adding/removing files.

## Layout

```
Aether/
  App/            AetherApp entry
  DesignSystem/   Theme tokens (colors, fonts, radii), LayerKind accents
  Models/         AppStore (state + actions), MusicData, SynthParams
  Components/     ArcKnob, EQ bars, curve/path builders, shared cards
  Screens/        Start, Stack hub, Lead Jam, Bass/Chords/Drums Inside,
                  Synth Editor, My Sounds, RootView navigation
  Audio/          AudioEngine → SynthCore (AVAudioEngine) + DSP primitives
  Resources/      Info.plist, Assets, Space Grotesk + JetBrains Mono fonts
```

## Fidelity notes

- Screens are reproduced on the mock's **393×852 design canvas** and scaled to the
  device's safe area (`ScreenChrome` in `RootView.swift`). The device background and
  the canvas share the same gradient, so the scale margins are seamless — and the
  mock's spacing and text wrapping reproduce exactly on any screen size.
- Fonts are bundled as variable TTFs (`SpaceGrotesk.ttf`, `JetBrainsMono.ttf`) and
  fall back to system fonts if unavailable.
- All curve/arc/ADSR/waveform geometry mirrors the prototype's exact math
  (`Components/Geometry.swift`).

## Audio engine

`Audio/SynthCore.swift` runs an `AVAudioSourceNode` render callback:

- **Wavetable oscillator** with unison/detune, morphing timbre (`DSP.swift`).
- **State-variable low-pass filter** (cutoff + resonance) per layer; the detail-screen
  EQ knobs and the editor's BRIGHT/CUT macros all feed the cutoff.
- **Per-voice ADSR**, soft-saturation **drive**, and a **shimmer reverb**
  (Schroeder reverb + octave-up feedback) fed by the SPACE / MIX macros.
- **Procedural drum synth** (kick/snare/hat) — drums are sample-based in spirit but
  synthesized here since no audio assets ship with the handoff.
- A **transport/sequencer** clocks 16th notes at the project BPM and plays the bass
  line, chord progression, and drum pattern; the **Lead** is performed live on the
  pad grid. UI note events hand off to the audio thread via a spinlock-guarded queue.

The four macro knobs map to: BRIGHT → filter brightness, MOVEMENT → wavetable LFO,
SPACE → reverb send, DRIVE → saturation.

### Self-test

Launching with `-audiotest` renders 2s offline and logs output RMS/peak/NaN —
a deterministic check that the graph produces stable, non-clipping audio:

```sh
xcrun simctl launch <device> app.neun.aether -audiotest
# AETHER_SELFTEST rms=0.323 peak=0.828 hasNaN=NO
```

`-route <start|stack|bass|chords|drums|lead|synth|sounds>` deep-links a screen
(QA aid; both flags are no-ops in normal use).
